# BNI SDLC Approvals System - Product Overview

## Purpose & Value Proposition
The BNI SDLC Approvals System is an internal enterprise web application designed to digitize and streamline the Software Development Life Cycle (SDLC) approval process for Bank Negara Indonesia (BNI). It replaces manual, paper-based approval workflows with an automated, trackable digital system that ensures compliance with IT governance requirements.

## Key Features & Capabilities

### Digital Initiative Management
- **Initiative Creation Wizard**: Streamlined interface for creating new SDLC projects with proper categorization (Project/Non-Project) and methodology selection (Waterfall/Agile)
- **Document Upload & Management**: Support for multiple document types (FS, BRD, Project Charter, ARF, FSD) with version control and approval tracking
- **Status Progression**: Automated status flow from Initiative Submitted through Go Live with proper governance checkpoints

### Approval Workflows
- **Configurable Approval Modes**: Sequential (ordered) or parallel approval processes per document type
- **Role-Based Approvers**: Support for multiple approver roles (IT BP, STA, PMO, IFC, ISA, DEV, APS, IFM, CISO)
- **Digital Signatures**: Integration with DocuSeal for legally binding e-signatures with audit trails
- **Real-time Status Tracking**: Live updates on approval progress with timeline visualization

### Document Management
- **PDF-Only Support**: Secure PDF upload with file validation and size limits (50MB max)
- **Version Control**: Multiple document versions with approval reset on new uploads
- **Signed Document Storage**: Persistent storage of digitally signed documents with certificates
- **Document Preview**: Built-in PDF viewer for document review

### User Management & Security
- **Role-Based Access Control**: Different permission levels (Admin, Approver, Initiator/Requester)
- **Secure Authentication**: Session-based authentication with secure cookie management
- **Audit Logging**: Comprehensive tracking of all user actions and system changes
- **User Administration**: Complete user management interface for administrators

### Dashboard & Analytics
- **Personal Dashboards**: Individual workspaces showing assigned tasks and pending approvals
- **Project Overview**: Executive-level metrics and project status summaries
- **Timeline Visualization**: Visual progress tracking through SDLC stages
- **Notification System**: Email notifications for approval assignments and status changes

## Target Users & Use Cases

### Primary Users
- **Business Users/SKTI (Requesters)**: Create initiatives, upload documents, track approval status, respond to feedback
- **IT Approvers**: Review and approve documents based on their expertise area (PMO, ISA, DEV, etc.)
- **System Administrators**: Manage users, configure approval workflows, maintain system settings

### Key Use Cases
1. **Initiative Submission**: Business users submit new project initiatives with required documentation (FS, BRD)
2. **Document Approval**: Configured approvers review and digitally sign documents in sequential or parallel workflows
3. **Status Progression**: Automatic project status advancement based on completed approvals and governance checkpoints
4. **Audit & Compliance**: Complete audit trails for regulatory compliance and internal governance
5. **Document Retrieval**: Access to signed documents and approval history for reference and compliance

## Business Value
- **Process Efficiency**: Reduces manual processing time and eliminates paper-based workflows
- **Compliance Assurance**: Ensures all required approvals are obtained before project progression
- **Transparency**: Real-time visibility into approval status and project progress
- **Audit Readiness**: Complete digital audit trails for regulatory and internal compliance
- **Cost Reduction**: Eliminates printing, physical storage, and manual tracking costs
